一、建议先修改 composer 的全局配置文件为国内镜像


因为composer服务在国外，在国内用composer时会比较慢，建议把composer服务配置成国内的，打开命令行窗口（windows用户）或控制台（Linux、Mac 用户）并执行如下命令：

composer config -g repo.packagist composer https://packagist.phpcomposer.com


上述命令将会在当前项目中的 composer.json 文件的末尾自动添加镜像的配置信息（你也可以自己手工添加）：

"repositories": {
    "packagist": {
        "type": "composer",
        "url": "https://packagist.phpcomposer.com"
    }
}


二、获取laravel-ide-helper扩展包


方法一、首先打开控制台，输入cd命令切换到项目根目录下，使用 composer 命令获取扩展包：

composer require barryvdh/laravel-ide-helper


方法二、找到项目composer.json，在require的值里面添加 "barryvdh/laravel-ide-helper":"dev-master"，然后执行 composer update 命令

"require": {
    "php": ">=7.0.0",
    "fideloper/proxy": "~3.3",
    "laravel/framework": "5.5.*",
    "laravel/tinker": "~1.0"，
    "barryvdh/laravel-ide-helper": "^2.5"
},


三、打开项目config/app.php 于providers添加如下一行


Barryvdh\LaravelIdeHelper\IdeHelperServiceProvider::class,


四、执行 php artisan ide-helper:generate 命令


最后会在根目录生成一个_ide_helper.php文件。
